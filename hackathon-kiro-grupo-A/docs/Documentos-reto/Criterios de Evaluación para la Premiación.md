# Criterios de Evaluación para la Premiación

Para que enfoquen sus esfuerzos estratégicamente, estos son los 5 puntos que el jurado valorará:

## 1. 📐 Calidad de la Especificación

- ¿Las user stories son claras, completas y verificables?
- ¿Los criterios de aceptación son específicos y medibles?
- ¿El diseño técnico documenta decisiones clave (arquitectura, dependencias, restricciones)?
- ⚠ **Nota vital:** Se evaluará el artefacto generado directamente en Kiro, no en documentos externos.

## 2. 💼 Relevancia para el Negocio

- ¿Resuelve un problema real y prioritario del negocio asegurador colombiano?
- ¿El impacto potencial es claro (reducción de costos, mejora en la experiencia del cliente, eficiencia operativa)?
- ¿Considera el contexto regulatorio y el mercado local?

## 3. ⚙ Uso Efectivo de Kiro y Spec-Driven Development

- ¿El equipo utilizó Kiro como herramienta central para especificar y construir?
- ¿Se evidencia un flujo claro de Especificación → Diseño → Código (evitando el código improvisado)?
- ¿Las especificaciones guiaron las decisiones de implementación de forma trazable?

## 4. 💻 Prototipo Funcional

- ¿El prototipo demuestra la funcionalidad principal descrita en las especificaciones?
- ¿Es ejecutable o demostrable en vivo?
- ¿El código es limpio, está organizado y alineado con lo que se especificó?

## 5. 🎤 Presentación y Colaboración

- ¿El pitch es claro, conciso y convincente (máximo 5 minutos)?
- ¿Se evidencia una colaboración real entre los PO (Product Owner)?
- ¿El equipo responde las preguntas del jurado con solidez y dominio del tema?
